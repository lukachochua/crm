# CRM Screenshot Walkthrough (Role-by-Role, Very Detailed)

Use this checklist to capture screenshots in the exact order. Each step indicates the page and the recommended capture moment.

## Common Screenshot Rules
- Capture after fields are filled but before saving (unless noted).
- Include the full page with navigation visible.
- Use consistent browser size and zoom.

---
# Sales Role

1) Login
   - Capture the /admin login page.

2) Customers List
   - Clients & Assets -> Customers list view.

3) Create Customer (Form Filled)
   - Customer create form with required fields filled.

4) Customer Saved
   - Customer edit page after save.

5) Vehicles List
   - Clients & Assets -> Vehicles list view.

6) Create Vehicle (Form Filled)
   - Vehicle create form with required fields filled.

7) Vehicle Saved
   - Vehicle edit page after save.

8) Applications List
   - Sales -> Applications list view.

9) Create Application (Form Filled)
   - Application create form with status=new and requested_at set.

10) Application Saved
   - Application edit page after save.

11) Application Status Change
   - Application edit page with status set to reviewed (before save).

12) Orders List
   - Sales -> Orders list view.

13) Create Order (Form Filled)
   - Order create form with customer + optional application selected.

14) Order Saved
   - Order edit page after save.

15) Order Status Change
   - Order edit page with status set to confirmed (before save).

16) Reservations List
   - Sales -> Reservations list view.

17) Create Reservation (Form Filled)
   - Reservation create form with order + vehicle selected.

18) Reservation Saved
   - Reservation edit page after save.

19) Reservation Status Change
   - Reservation edit page with status set to fulfilled (before save).

20) Invoices List (Read-only)
   - Invoicing -> Invoices list view.
